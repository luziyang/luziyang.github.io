发表于 2017-03-01

1、MVC
MVC是为了将模型和业务彻底分离.
2、什么是Spring?
Spring是个Java企业级应用的开源开发框架。Spring主要用来开发Java应用，但是有些扩展是针对构建J2EE平台的web应用。Spring框架目标是简化Java企业级应用开发，并通过pojo
为基础的编程模型促进良好的编程习惯。
简单来说。Spring就是一个轻量级的控制反转(IOC)和面向切面(AOP)编程的容器框架。
轻量 : 从大小与开销两方面而言Spring都是轻量的。此外Spring是非入侵式的 : 使用Spring，我们的类还是pojo类，完全不用实现spring的类和接口等
也就说，使用Spring中的应用对象不需要继承或实现类和接口

IOC : Spring通过控制反转来实现松耦合，当应用了IOC，一个对象依赖的其他对象会通过被动的方式传递进来，而不是这个对象创建或者查找依赖对象
基本上就是对象不用自己手动创建和管理，完全由容器管理，我们只管用就行了

AOP : Spring 提供了面型切面编程的支持，AOP将与程序无关的内容分离提取，应用对象只实现其应该做的–完成业务逻辑–仅此而已。它们并不负责其他系统级关注点，例如日志或事务的支持。
AOP将与业务无关的逻辑横切进真正的逻辑中

框架 ：Spring可以将简单的组件配置、组合成复杂的应用。在Spring中，应用对象被声明式地组合，典型的是在一个XML文件里。Spring也提供了很多基本功能(事务管理、持久化框架集成等等)，
而用户就用跟多的时间去开发应用逻辑。

所有的Spring的这些特征都能帮助我们能够更干净、更可管理、并且易于测试的代码。它们也为Spring中的各种模块提供了基础支持

借助Spring,荣国依赖注入，AOP应用，面向接口编程，来降低业务组件之间的耦合度，增强系统的扩展性。 让已有的技术和框架更加易用。
利用其对hibernate的SessionFactory、事务管理的封装，更简洁的应用hibernate. Spring并不完全依赖于Spring,开发者可自由选用Spring框架的部分或全部
利用AOP思想，集中处理业务逻辑，减少重复代码，构建优雅的解决方案。 低侵入式设计，代码污染极低。
3、什么是SpringBeans
SpringBeans是那些Spring应用的主干的java对象。它们被Spring IOC容器初始化，装配和管理。这些beans通过容器中配置的元数据创建，比如、以XML文件中的的形式定义。
Spring框架的Beans都是单件Beans。在bean tag中有个属性singleton，如果它被赋予TRUE，bean就是单例，否则就是一个prototype bean。默认是TRUE，所以所有在Spring框架中的Beans缺省都是单例。
4、Spring使用什么创建对象?
通过反射创建对象
5、Spring通过什么来管理对象?
Spring会通过反射创建对象，然后将对象存储在集合当中，并给每一个对象一个Key/name.当我们需要某个对象时，就可以直接通过key/name来获取对象，而且Spring在返回对象时还会通过DI将我们所需要的属性直接注入。
6、Spring中的单例bean是线程安全的吗?
不是，因为是单例的所以和servlet相似，不是线程安全的
7、简单说下Spring中bean对象的生命周期?
默认情况下在Spring荣丰器被创建时，所有的对象会被创建，在容器销毁时所有的对象会被销毁。但是当配置了lazy-ini为TRUE时，在容器创建时，对象不会被创建，只有在需要使用对象时才会创建对象。当然我们可以为每个
bean制定对应的init和destory方法。
8、Spring的bean装配方式都有哪些?
自动装配 : 无论是使用配置文件(autowire)还是使用注解(@autowire)，都可以配置为自动装配。配置方式为按类型装配(ByType),和按名称装配(ByName);
手动装配 ：无论是使用配置文件(ref)还是使用注解(@resource)都可以直接指定装配bean的名字。
